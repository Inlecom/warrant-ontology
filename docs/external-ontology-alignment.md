# WARRaNT KG Ontology — External Ontology Alignment (Informative)

Version 0.9-poc | 2026-06-05

**Status: INFORMATIVE ONLY**

In v0.1.0 this alignment is informative. No `owl:equivalentClass`, `owl:sameAs`, or property substitution assertions are added to the ontology files. Normative alignment is deferred to v0.2.0 pending consortium agreement.

---

## 1. W3C SOSA/SSN

**Sensor, Observation, Sample, Actuator (SOSA) and Semantic Sensor Network (SSN)**
Namespace: `http://www.w3.org/ns/sosa/`, `http://www.w3.org/ns/ssn/`

| WARRaNT term | External term | Semantic fit | Notes |
|---|---|---|---|
| `obs:Observer` | `sosa:Sensor` | close | SOSA:Sensor is restricted to physical sensing; obs:Observer includes analytical services |
| `obs:Measurement` | `sosa:Observation` | close | sosa:Observation includes feature-of-interest; WARRaNT uses obs:hasDependabilityMetric instead |
| `obs:Metric` | `ssn:Property` / `sosa:ObservableProperty` | close | WARRaNT Metric is at component level; SOSA ObservableProperty is at feature level |
| `obs:hasValue` | `sosa:hasSimpleResult` | exact | Both carry a scalar result |
| `obs:hasTimestamp` | `sosa:resultTime` | exact | Both carry the timestamp of the result |
| `obs:VirtualSensor` | `sosa:Sensor` | close | Software-only nature not captured by SOSA; note needed |
| `obs:AnalyticalService` | `sosa:Sensor` | partial | SOSA has no direct equivalent for computational analytical services |
| `obs:DetectionEvent` | — | none | WARRaNT-specific; no direct SOSA/SSN equivalent |

**Gaps:** SOSA has no concept equivalent to `obs:DetectionEvent` or `obs:AnalyticalService` as a computational reasoning agent. These are WARRaNT-specific.

**Recommendation:** Future releases should allow `obs:Observer` to be aligned with `sosa:Sensor` via `rdfs:subClassOf` after consortium review.

## 2. QUDT

**Quantities, Units, Dimensions, Types**
Namespace: `http://qudt.org/schema/qudt/`

| WARRaNT term | External term | Semantic fit | Notes |
|---|---|---|---|
| `obs:hasUnit` | `qudt:unit` | close | WARRaNT uses xsd:string; QUDT uses specific unit IRIs |
| `obs:hasValue` | `qudt:numericValue` | close | Both carry a numeric scalar |

**Recommendation:** Future releases should allow `obs:hasUnit` to accept QUDT unit IRIs directly (e.g. `qudt:MilliSecond`, `qudt:Percent`). This would improve interoperability with external data pipelines.

## 3. PROV-O

**W3C Provenance Ontology**
Namespace: `http://www.w3.org/ns/prov#`

| WARRaNT term | External term | Semantic fit | Notes |
|---|---|---|---|
| `obs:AnalyticalService` | `prov:SoftwareAgent` | close | AnalyticalService is a software agent producing provenance-tracked outputs |
| `obs:DetectionEvent` | `prov:Activity` | close | A detection event is an activity that generates a Deviation entity |
| `assr:AssuranceDegradation` | `prov:Entity` | partial | The degradation is an entity generated by a detection activity |

**Note:** Provenance tracing of partner contributions (who created which ontology term, when) is out of scope for v0.1.0.

**Recommendation:** v0.2.0 should evaluate whether `assr:AssuranceDegradation` should carry `prov:wasGeneratedBy` linking to the `obs:DetectionEvent` that triggered it.

## 4. SAREF / SAREF4ENER

**Smart Appliances Reference Ontology**
Namespace: `https://saref.etsi.org/core/`, `https://saref.etsi.org/saref4ener/`

| WARRaNT term | External term | Semantic fit | Notes |
|---|---|---|---|
| `mit:MitigationAction` | `saref:Command` | partial | SAREF Command is targeted at smart appliance control; maritime operational context differs significantly |
| `davom:ControlAction` | `saref:Command` | partial | Similar partial fit |
| `mit:MachineActionableAction` | `saref:Command` | partial | Closest fit, but SAREF is appliance-focused |

**Gaps:** SAREF is designed for smart home/grid appliances and does not cover maritime vessel operations, remote operation, or safety-critical contexts.

**Recommendation:** Alignment with SAREF is low priority. Monitor ETSI SAREF4NEMO (maritime) developments.

---

## 5. GSN / SACM

**Goal Structuring Notation / Structured Assurance Case Metamodel**
Namespace: `http://www.omg.org/spec/SACM/`

| WARRaNT term | External term | Semantic fit | Notes |
|---|---|---|---|
| `assr:Evidence` | `gsn:Evidence` | close | Both represent evidence supporting an assurance claim |
| `assr:AssuranceScore` | `gsn:Goal` | partial | The score is a quantitative claim about dependability; GSN Goal is a qualitative claim |

**Note:** Formal assurance case representation using GSN/SACM is planned as a future WARRaNT deliverable (AssuranceOPS — T1.4, DNV lead).

**Recommendation:** v0.3.0 should model the link between `assr:AssuranceScore` and a GSN/SACM assurance case goal.
